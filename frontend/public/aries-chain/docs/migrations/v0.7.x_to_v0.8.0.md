# Cosmos EVM v0.7.x -> v0.8.0 Migration

v0.8.0 removes the deprecated `contrib/x/precisebank` module and standardizes on 18-decimal EVM gas-token configurations.

## Key change

- `contrib/x/precisebank` is no longer shipped in this repository.
- Non-18-decimal gas-token chains are not supported in v0.8.0.

## Required actions

1. Remove all `contrib/x/precisebank` imports from your app and tests.
2. Remove precisebank keeper/module wiring from app construction:
   - `PreciseBankKeeper` fields/getters
   - `precisebank.NewKeeper(...)`
   - `precisebank.NewAppModule(...)`
   - begin/end/init module ordering entries for precisebank
3. Replace call sites that previously relied on precisebank translation with native 18-decimal flows.
4. Ensure your EVM coin config uses 18 decimals (`EvmCoinInfo.Decimals = 18`).

## Operator guidance

- If your chain still depends on non-18-decimal behavior, do not upgrade to v0.8.0 yet.
- Migrate token denomination/decimals strategy first, then upgrade.
